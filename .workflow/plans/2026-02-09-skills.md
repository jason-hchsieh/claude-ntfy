# Plan: Claude Code Skills — Setup Server & Test Notification

## Overview

Add two Claude Code skills to the plugin:
1. **setup-server** — Guides users through setting up an ntfy server (Docker) and configuring env vars
2. **test-notification** — Sends a test notification to verify the setup works

Skills are auto-discovered markdown files in `skills/<name>/SKILL.md`. They activate automatically when Claude detects a matching task context.

## Architecture

```
skills/
├── setup-server/
│   └── SKILL.md          # Guides through ntfy server setup
└── test-notification/
    └── SKILL.md          # Sends test notification to verify config
```

## Tasks

### Task 1: Create setup-server skill
No dependencies

- **Implement:** `skills/setup-server/SKILL.md`
- Covers:
  - Check if Docker is available
  - Start ntfy via docker compose (using `docker/docker-compose.yml`)
  - Verify ntfy is running (curl health check)
  - Guide user to set `NTFY_TOPIC`, `NTFY_SERVER_URL`, `NTFY_TOKEN` env vars
  - Update `.mcp.json` env section if needed

### Task 2: Create test-notification skill
No dependencies (parallel with Task 1)

- **Implement:** `skills/test-notification/SKILL.md`
- Covers:
  - Read current env var config
  - Send a test notification via curl to `$NTFY_SERVER_URL/$NTFY_TOPIC`
  - Verify delivery (check response status)
  - Suggest subscribing to the topic via ntfy app/web UI

### Task 3: Update plugin manifest & CLAUDE.md
Depends on: Task 1, Task 2

- Update CLAUDE.md to document new skills
- Verify plugin auto-discovery picks up the skills directory

## Dependency Graph

```
Task 1 (setup-server) ──┐
                         ├──→ Task 3 (manifest + docs)
Task 2 (test-notification)─┘
```
